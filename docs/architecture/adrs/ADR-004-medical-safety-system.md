# ADR-004: Архитектура системы медицинской безопасности

## Статус

Accepted

## Контекст

Aibolit AI — AI-система, предоставляющая медицинскую информацию пациентам. Это создаёт уникальные требования безопасности:

1. **Риск для здоровья** — некорректная информация может привести к самолечению, задержке обращения за помощью или причинению вреда здоровью.
2. **Экстренные состояния** — пациент может описывать симптомы, угрожающие жизни (инфаркт, инсульт, анафилаксия).
3. **Правовые требования** — AI не может ставить диагнозы, назначать лечение, выписывать рецепты (ФЗ-323).
4. **Prompt injection** — злоумышленник может попытаться заставить AI обойти safety-ограничения.
5. **Медицинские данные** — специальная категория ПД по 152-ФЗ, требует повышенной защиты.

Система должна обеспечивать баланс между **полезностью** (информативные ответы) и **безопасностью** (отсутствие вреда).

## Решение

Реализована трёхуровневая система медицинской безопасности с компонентами на уровне ввода, AI-генерации и вывода.

### Архитектура

```
Пользовательский ввод
         |
         v
+------------------+
| 1. INPUT SAFETY  |
|   Sanitization   |  ← Фильтрация prompt injection
|   Red Flags      |  ← Детекция экстренных состояний
+--------+---------+
         |
         v
+------------------+
| 2. AI SAFETY     |
|   System Prompt  |  ← Правила поведения AI-врача
|   --max-turns 3  |  ← Ограничение рекурсии
|   Timeout 120s   |  ← Защита от зависания
+--------+---------+
         |
         v
+------------------+
| 3. OUTPUT SAFETY |
|   Disclaimers    |  ← 8 типов дисклеймеров
|   Emergency UI   |  ← Экстренное уведомление
|   SSE meta event |  ← Метаданные безопасности
+------------------+
         |
         v
   Ответ пользователю
```

### Компонент 1: Input Safety

#### 1.1 Sanitization (фильтрация prompt injection)

**Файл:** `web/backend/services/chat_service.py` → `sanitize_user_input()`

```python
def sanitize_user_input(text: str) -> str:
    # 1. Удаление paттернов "ignore previous instructions"
    text = re.sub(
        r'(?i)(ignore\s+(all\s+)?(previous|above|prior)\s+(instructions?|prompts?|rules?))',
        '[removed]', text,
    )
    # 2. Удаление паттернов "you are now / act as / system:"
    text = re.sub(
        r'(?i)(you\s+are\s+now|act\s+as|pretend\s+(to\s+be|you\s+are)|system\s*:)',
        '[removed]', text,
    )
    # 3. Удаление HTML-тегов
    text = re.sub(r'<[^>]{0,50}>', '', text)
    # 4. Ограничение длины
    return text[:5000].strip()
```

**Принцип:** Удаление известных паттернов injection, не блокирование пользователя. Пациент может использовать слова вроде "ignore" в медицинском контексте, поэтому фильтруются только полные паттерны.

#### 1.2 Red Flag Detection (детекция экстренных состояний)

**Файл:** `src/safety/red_flags.py` → `RedFlagDetector`

```yaml
Принцип работы:
  1. Компилируются regex-паттерны для 20+ экстренных состояний
  2. На каждом входящем сообщении все паттерны проверяются
  3. Обнаруженные flags классифицируются по urgency (1-5)
  4. Urgency 5 (IMMEDIATE): рекомендация вызвать 103/112
  5. Urgency 4 (VERY_HIGH): рекомендация срочно обратиться к врачу

Категории:
  - Кардиологические (боль в груди, обморок, одышка)
  - Неврологические (односторонняя слабость, потеря зрения, судороги)
  - Аллергические (анафилаксия, отёк Квинке)
  - Психиатрические (суицидальные мысли)
  - Абдоминальные (рвота кровью, мелена, перитонит)
  - Акушерские (кровотечение при беременности)
  - Инфекционные (менингит)
  - Эндокринные (гипергликемия >20)
  - Сосудистые (тромбоз глубоких вен)
  - Урологические (задержка мочи)
  - Онкологические (впервые выявленные образования)
  - Хирургические (аппендицит)
```

**Ключевое ограничение:** Regex-паттерны на русском языке. Пациент может описать симптомы нестандартно — false negative rate не измерен (GAP-003).

### Компонент 2: AI Safety (System Prompt)

**Файл:** `web/backend/services/chat_service.py` → `build_system_prompt()`

System prompt содержит:
1. **Роль:** "Ты — AI-{специализация} в клинике Aibolit"
2. **Ограничения:** "НЕ ставишь диагнозы, НЕ назначаешь лечение"
3. **Контекст пациента:** Данные из медкарты (аллергии, диагнозы, препараты)
4. **Правила формулировок:**
   - Использовать: "ВОЗМОЖНО", "МОЖЕТ БЫТЬ СВЯЗАНО С", "РЕКОМЕНДУЕТСЯ ОБСУДИТЬ С ВРАЧОМ"
   - Не использовать: "У вас [диагноз]", "Вам нужно принимать [лекарство]"
5. **Поведение:** Задавать уточняющие вопросы, быть конкретным и профессиональным
6. **Язык:** Русский

**Ограничения CLI:**
- `--max-turns 3` — ограничивает рекурсию tool use (если Claude решит использовать инструменты)
- Timeout 120 секунд — защита от зависания

### Компонент 3: Output Safety

#### 3.1 Disclaimers (дисклеймеры)

**Файл:** `src/safety/disclaimers.py`

8 типов дисклеймеров, подбираемых по контексту:

| Тип | Когда применяется |
|-----|------------------|
| GENERAL | Каждый ответ AI |
| DIAGNOSIS | При анализе симптомов |
| TREATMENT | При упоминании лечения |
| MEDICATION | При упоминании лекарств |
| LAB_ANALYSIS | При расшифровке анализов |
| IMAGING | При анализе изображений |
| EMERGENCY | При экстренных состояниях (всегда первый) |
| CHILDREN | При педиатрической консультации |

#### 3.2 SSE Meta Event

При создании чата и при каждом сообщении клиент получает SSE-событие `meta`:

```json
{
  "consultation_id": 42,
  "doctor": {"specialty_id": "cardiologist", "name": "AI-Кардиолог"},
  "disclaimer": "Информация предоставлена AI-ассистентом...",
  "red_flags": [
    {"category": "cardiac", "description": "Боль в груди", "urgency": 5, "action": "Вызовите скорую: 103"}
  ],
  "emergency": {
    "call": "103 / 112",
    "message": "Обнаружены признаки, требующие экстренной помощи!",
    "flags": [...]
  }
}
```

Frontend отображает:
- **Emergency alert** — красный баннер с номером скорой помощи
- **Red flags** — предупреждения с описанием и рекомендуемыми действиями
- **Disclaimer** — текст после каждого ответа AI

## Альтернативы

### 1. Guardrails AI (guardrails-ai)

**Плюсы:**
- Структурированная валидация вывода AI
- Готовые validators для medical content
- Input/output guards

**Минусы:**
- Дополнительная зависимость
- Latency overhead (дополнительный LLM-вызов)
- Не поддерживает CLI-based AI
- Избыточен для текущего scope

**Отклонено для MVP:** Overhead и несовместимость с CLI.

### 2. NeMo Guardrails (NVIDIA)

**Плюсы:**
- Программируемые safety rails
- Topical rails (ограничение темы)
- Moderation rails

**Минусы:**
- Сложная конфигурация (Colang)
- GPU-зависимость
- Не поддерживает CLI-based AI

**Отклонено:** Избыточная сложность и hardware requirements.

### 3. Ручная валидация вывода AI

**Плюсы:**
- Полный контроль
- Можно фильтровать запрещённые формулировки в ответе AI

**Минусы:**
- Regex на медицинском тексте — много false positives
- Задержка вывода (нужно собрать весь ответ перед отправкой)
- Несовместим со streaming

**Частично принято:** Рекомендуется как будущее улучшение для не-streaming endpoints.

## Последствия

### Положительные

1. **Многоуровневая защита** — input (sanitization + red flags) + AI (system prompt) + output (disclaimers + emergency).
2. **Real-time alerts** — экстренные состояния обнаруживаются до отправки к AI.
3. **Минимальная latency** — regex-детекция добавляет <1ms.
4. **Масштабируемость** — паттерны легко расширяются.
5. **Прозрачность** — дисклеймеры и red flags видны пользователю.

### Отрицательные

1. **Нет валидации вывода AI** — system prompt не гарантирует соблюдение правил.
2. **Regex-ограничения** — паттерны не покрывают все формулировки (перефразирования, сленг).
3. **Нет тестов** — false negative/positive rates не измерены (GAP-003).
4. **Нет ML-детектора** — простые regex вместо NLP-классификатора.
5. **Нет модуля живого врача** — эскалация ограничена рекомендацией позвонить в скорую (DEC-006).

### Метрики (планируемые)

```yaml
Red Flag Detection:
  - False Negative Rate: < 5% (экстренные состояния не пропускаются)
  - False Positive Rate: < 20% (допустимый уровень ложных тревог)
  - Coverage: 20+ immediate patterns, 6+ high patterns
  - Latency: < 1ms per message

Disclaimers:
  - Coverage: 100% ответов AI содержат дисклеймер
  - Type accuracy: Правильный тип дисклеймера для контента

Prompt Injection:
  - Detection Rate: > 95% known patterns
  - False Positive Rate: < 5% (легитимные медицинские запросы не блокируются)
```

### Roadmap улучшений

```yaml
P1 (до production):
  - [ ] Тесты red flags (все patterns, false negatives)
  - [ ] Тесты дисклеймеров (coverage)
  - [ ] Тесты prompt injection (OWASP LLM Top 10)

P2 (после production):
  - [ ] Output validation (проверка ответа AI на запрещённые формулировки)
  - [ ] ML-классификатор для red flags (NLP вместо regex)
  - [ ] A/B-тестирование дисклеймеров (readability)
  - [ ] Мониторинг false positive/negative rates
  - [ ] Canary tokens в system prompt

P3 (при масштабировании):
  - [ ] Модуль живого врача (эскалация через чат)
  - [ ] Guardrails AI для structured output validation
  - [ ] Multi-language support (не только русский)
```
