# Aibolit AI — User Stories

## Соглашения

- **Формат:** Как [роль], я хочу [действие], чтобы [результат]
- **Приоритеты:** CRITICAL > HIGH > MEDIUM > LOW
- **Статус:** Done (реализовано в MVP), Todo (запланировано)
- **Нумерация:** US-{Эпик}-{Номер}

---

## Эпик 1: Регистрация и авторизация

### US-AUTH-001: Регистрация пациента
- **Приоритет:** CRITICAL | **Статус:** Done
- **Как** пациент, **я хочу** зарегистрироваться, указав email, пароль, ФИО, дату рождения и базовую медицинскую информацию, **чтобы** получить доступ к AI-консультациям.

**Acceptance Criteria:**
- [ ] 3-шаговая форма регистрации: учётные данные -> персональные данные -> медицинская информация
- [ ] Валидация email (формат, уникальность)
- [ ] Пароль: минимум 8 символов
- [ ] После регистрации — автоматический login и редирект на Dashboard
- [ ] Данные сохраняются в БД с хэшированным паролем (SHA-256 + salt)

### US-AUTH-002: Вход в систему
- **Приоритет:** CRITICAL | **Статус:** Done
- **Как** зарегистрированный пациент, **я хочу** войти по email и паролю, **чтобы** получить доступ к своему профилю и истории консультаций.

**Acceptance Criteria:**
- [ ] Форма логина с email и паролем
- [ ] JWT-токен сохраняется в localStorage
- [ ] При невалидных данных — сообщение об ошибке
- [ ] Редирект на Dashboard после успешного входа
- [ ] Перенаправление неавторизованных пользователей на `/login`

### US-AUTH-003: Информированное согласие
- **Приоритет:** HIGH | **Статус:** Todo
- **Как** пациент, **я хочу** ознакомиться и принять информированное согласие при регистрации, **чтобы** понимать, что AI не заменяет врача и как обрабатываются мои данные.

**Acceptance Criteria:**
- [ ] Текст согласия включает: AI не ставит диагнозов, данные — специальная категория ПД, цели обработки
- [ ] Чекбокс «Я согласен» обязателен для завершения регистрации
- [ ] Дата и время согласия сохраняются в БД
- [ ] Возможность отозвать согласие в настройках

### US-AUTH-004: Refresh Token
- **Приоритет:** HIGH | **Статус:** Todo
- **Как** пациент, **я хочу**, чтобы сессия автоматически продлевалась, **чтобы** не вводить пароль каждые 30 минут.

**Acceptance Criteria:**
- [ ] Access token: 30 минут
- [ ] Refresh token: 7 дней
- [ ] Автоматическое обновление access token при его истечении
- [ ] Refresh token хранится в httpOnly cookie (не в localStorage)
- [ ] При истечении refresh token — редирект на login

## Эпик 2: Профиль пациента

### US-PROFILE-001: Просмотр Dashboard
- **Приоритет:** HIGH | **Статус:** Done
- **Как** пациент, **я хочу** видеть обзорный Dashboard с основными показателями здоровья, **чтобы** быстро оценить своё состояние.

**Acceptance Criteria:**
- [ ] Последние витальные показатели (давление, пульс, температура, вес)
- [ ] Активные диагнозы
- [ ] Текущие назначения (лекарства)
- [ ] Недавние консультации
- [ ] Быстрые действия: начать чат, добавить витальные, записать анализы

### US-PROFILE-002: Управление витальными показателями
- **Приоритет:** HIGH | **Статус:** Done
- **Как** пациент, **я хочу** вносить витальные показатели (давление, пульс, температура, вес, рост, сахар, SpO2), **чтобы** отслеживать динамику и предоставлять данные AI-врачу.

**Acceptance Criteria:**
- [ ] Форма ввода витальных с валидацией диапазонов
- [ ] История витальных в виде таблицы и/или графиков
- [ ] Дата и время для каждого измерения
- [ ] Цветовая индикация: норма (зелёный), отклонение (жёлтый), критическое (красный)

### US-PROFILE-003: Управление лабораторными анализами
- **Приоритет:** HIGH | **Статус:** Done
- **Как** пациент, **я хочу** вносить результаты лабораторных анализов, **чтобы** получать их расшифровку от AI.

**Acceptance Criteria:**
- [ ] Форма ввода: название теста, значение, единицы, дата
- [ ] Сравнение с референсными значениями
- [ ] Тренды показателей во времени
- [ ] Кнопка «Расшифровать» для отправки в модуль диагностики

### US-PROFILE-004: Управление диагнозами
- **Приоритет:** MEDIUM | **Статус:** Done
- **Как** пациент, **я хочу** вести список своих диагнозов (код МКБ-10, дата, статус), **чтобы** AI-врач учитывал их при консультации.

**Acceptance Criteria:**
- [ ] Добавление диагноза: название, МКБ-10 код, дата постановки, статус (активный/в ремиссии/снят)
- [ ] Редактирование и архивирование диагнозов
- [ ] Поиск по МКБ-10 при добавлении

### US-PROFILE-005: Управление назначениями
- **Приоритет:** MEDIUM | **Статус:** Done
- **Как** пациент, **я хочу** вести список лекарств, которые принимаю, **чтобы** AI-врач учитывал их и проверял взаимодействия.

**Acceptance Criteria:**
- [ ] Добавление: название препарата, дозировка, частота, дата начала/окончания
- [ ] Статус: активное/завершённое
- [ ] Быстрая проверка взаимодействий для всех активных назначений

## Эпик 3: AI-консультации

### US-CHAT-001: Начало чата с AI-врачом
- **Приоритет:** CRITICAL | **Статус:** Done
- **Как** пациент, **я хочу** начать чат с AI-врачом выбранной специализации, **чтобы** получить информационную оценку своих симптомов.

**Acceptance Criteria:**
- [ ] Выбор из 35 специализаций (с описанием каждой)
- [ ] Новый сеанс чата создаётся с уникальным session_id
- [ ] Первое сообщение AI содержит приветствие, роль и дисклеймер
- [ ] AI знает специализацию и ведёт диалог в её рамках

### US-CHAT-002: Мультитерновый диалог
- **Приоритет:** CRITICAL | **Статус:** Done
- **Как** пациент, **я хочу** вести многоходовый диалог с AI-врачом, **чтобы** детально описать симптомы и получить полную информационную оценку.

**Acceptance Criteria:**
- [ ] SSE streaming: ответ AI появляется по мере генерации (не ждать полного ответа)
- [ ] Контекст сохраняется между сообщениями (через `--session-id`)
- [ ] Индикатор «AI печатает...»
- [ ] Минимум 20 сообщений в одном диалоге без потери контекста
- [ ] Каждый ответ AI содержит дисклеймер

### US-CHAT-003: Обнаружение red flags
- **Приоритет:** CRITICAL | **Статус:** Done (частично)
- **Как** пациент, **я хочу**, чтобы AI обнаружил опасные симптомы и предупредил меня, **чтобы** я мог своевременно вызвать скорую.

**Acceptance Criteria:**
- [ ] Red flags детектируются в тексте сообщения пациента (regex-паттерны)
- [ ] При обнаружении — предупреждение с рекомендацией вызвать 103/112
- [ ] Категории: кардио, неврология, аллергия, психиатрия, абдоминальные, акушерские, инфекционные
- [ ] False negative rate < 5% (тесты — US-SAFE-002)
- [ ] Логирование: `red_flag_detected`, `doctor_escalation_triggered`

### US-CHAT-004: История чатов
- **Приоритет:** HIGH | **Статус:** Done
- **Как** пациент, **я хочу** видеть историю прошлых чатов с AI-врачами, **чтобы** вернуться к рекомендациям и продолжить диалог.

**Acceptance Criteria:**
- [ ] Список чатов с датой, специализацией, первым сообщением
- [ ] Просмотр полной истории диалога
- [ ] Возможность продолжить существующий чат

### US-CHAT-005: Выбор врача через навигацию
- **Приоритет:** HIGH | **Статус:** Done
- **Как** пациент, **я хочу** выбрать специализацию AI-врача из наглядного каталога, **чтобы** быстро найти нужного специалиста.

**Acceptance Criteria:**
- [ ] Страница ConsultDoctor с карточками специализаций
- [ ] Иконка, название, краткое описание для каждой специализации
- [ ] Поиск/фильтрация по названию
- [ ] Клик на специализацию — переход в чат

## Эпик 4: Диагностика

### US-DIAG-001: Расшифровка лабораторных анализов
- **Приоритет:** HIGH | **Статус:** Done
- **Как** пациент, **я хочу** получить AI-расшифровку своих анализов крови, **чтобы** понять, что означают отклонения.

**Acceptance Criteria:**
- [ ] Ввод: список тестов с значениями и единицами
- [ ] Вывод: интерпретация каждого показателя (норма / повышен / понижен), severity
- [ ] Обнаруженные паттерны (patterns_detected)
- [ ] Общее резюме (summary)
- [ ] Дисклеймер типа `lab_analysis`

### US-DIAG-002: Оценка витальных показателей
- **Приоритет:** HIGH | **Статус:** Done
- **Как** пациент, **я хочу** получить AI-оценку моих витальных показателей, **чтобы** понять, требуется ли обратиться к врачу.

**Acceptance Criteria:**
- [ ] Ввод: давление, пульс, температура, SpO2, ЧДД
- [ ] Вывод: алерты (если значения вне нормы), severity, интерпретация
- [ ] Red flags при критических значениях
- [ ] Дисклеймер типа `general`

### US-DIAG-003: Расчёт скорости клубочковой фильтрации (СКФ)
- **Приоритет:** MEDIUM | **Статус:** Done
- **Как** пациент с заболеванием почек, **я хочу** рассчитать СКФ по формуле CKD-EPI, **чтобы** понять стадию ХБП.

**Acceptance Criteria:**
- [ ] Ввод: креатинин, возраст, пол
- [ ] Вывод: значение eGFR, стадия (G1-G5), рекомендация
- [ ] Цветовая индикация стадии

### US-DIAG-004: Оценка кардиоваскулярного риска
- **Приоритет:** MEDIUM | **Статус:** Done
- **Как** пациент, **я хочу** узнать свой 10-летний риск сердечно-сосудистых заболеваний, **чтобы** принять меры профилактики.

**Acceptance Criteria:**
- [ ] Ввод: возраст, пол, давление, холестерин, курение, диабет
- [ ] Вывод: процент риска, категория (низкий/умеренный/высокий/очень высокий), цвет, рекомендации
- [ ] Дисклеймер типа `diagnosis`

## Эпик 5: Лекарства

### US-DRUGS-001: Поиск препаратов
- **Приоритет:** HIGH | **Статус:** Done
- **Как** пациент, **я хочу** найти информацию о лекарственном препарате, **чтобы** узнать показания, противопоказания и побочные эффекты.

**Acceptance Criteria:**
- [ ] Поиск по названию (OpenFDA)
- [ ] Карточка препарата: действующее вещество, показания, противопоказания, побочные эффекты
- [ ] Дисклеймер типа `medication`

### US-DRUGS-002: Проверка взаимодействий лекарств
- **Приоритет:** HIGH | **Статус:** Done
- **Как** пациент, принимающий несколько препаратов, **я хочу** проверить их совместимость, **чтобы** избежать опасных взаимодействий.

**Acceptance Criteria:**
- [ ] Ввод: список из 2+ препаратов
- [ ] Вывод: обнаруженные взаимодействия с severity (major/moderate/minor)
- [ ] Для каждого взаимодействия: описание, рекомендация
- [ ] Дисклеймер типа `medication`

### US-DOCS-001: Просмотр медицинских документов
- **Приоритет:** MEDIUM | **Статус:** Done
- **Как** пациент, **я хочу** просматривать свои медицинские документы (направления, заключения), **чтобы** иметь к ним доступ в одном месте.

**Acceptance Criteria:**

- **Как** пациент, **я хочу** получить сформированный документ по итогам AI-консультации, **чтобы** показать его врачу.

**Acceptance Criteria:**
- [ ] Генерация сводки консультации (симптомы, предположения, рекомендации)
- [ ] Дисклеймер: документ не имеет юридической силы
- [ ] Экспорт / копирование текста

## Эпик 7: Безопасность и комплаенс

### US-SAFE-001: Шифрование медицинских данных
- **Приоритет:** CRITICAL | **Статус:** Todo
- **Как** пациент, **я хочу**, чтобы мои медицинские данные были зашифрованы в базе данных, **чтобы** в случае утечки БД данные были защищены.

**Acceptance Criteria:**
- [ ] Шифрование полей: diagnosis_code, diagnosis_text, lab_results, medication_list, allergy_info
- [ ] Алгоритм: AES-256 или Fernet (symmetric)
- [ ] Ключ шифрования в переменной окружения (не в коде)
- [ ] Миграция существующих данных в зашифрованный формат
- [ ] Расшифровка прозрачна для API (пользователь не замечает)

### US-SAFE-002: Тесты Red Flags
- **Приоритет:** HIGH | **Статус:** Todo
- **Как** команда разработки, **мы хотим** автоматические тесты детектора red flags, **чтобы** гарантировать false negative rate < 5%.

**Acceptance Criteria:**
- [ ] Unit-тесты для каждой категории: cardiac (5+ тестов), neurological (5+), psychiatric (3+), abdominal (3+), obstetric (2+), allergic (3+), endocrine (2+), vascular (2+), urological (2+), oncological (2+), surgical (2+)
- [ ] Тесты false positive: безопасные тексты не вызывают alarm (10+ тестов)
- [ ] Тесты вариаций: разные способы описания одного red flag (5+ тестов)
- [ ] Метрика FN rate вычисляется: количество пропущенных / всего red flag кейсов < 5%
- [ ] CI/CD: тесты выполняются при каждом коммите

### US-SAFE-003: Тесты дисклеймеров
- **Приоритет:** HIGH | **Статус:** Todo
- **Как** команда разработки, **мы хотим** автоматические тесты присутствия дисклеймеров, **чтобы** гарантировать 100% coverage.

**Acceptance Criteria:**
- [ ] Тест: каждый из 8 типов дисклеймеров существует и непустой
- [ ] Тест: функция `get_disclaimer()` возвращает корректный текст для каждого типа
- [ ] Integration-тест: ответ API `/chat/send` содержит дисклеймер
- [ ] Тест: при red flag — emergency дисклеймер
- [ ] Тест: при педиатрии — children дисклеймер

### US-SAFE-004: Аудит-логирование
- **Приоритет:** HIGH | **Статус:** Todo
- **Как** администратор системы, **я хочу** видеть аудит-лог всех медицинских событий, **чтобы** обеспечить прозрачность и отследить инциденты.

**Acceptance Criteria:**
- [ ] Логирование всех CRUD-операций с данными пациента
- [ ] Логирование медицинских событий: ai_consultation_started, ai_diagnosis_suggested, red_flag_detected, doctor_escalation_triggered
- [ ] Маскирование чувствительных полей (password, token, diagnosis_code и т.д.)
- [ ] Файлы: audit.log (общий), medical.log (медицинские события)
- [ ] Retention: 5 лет для медицинских логов

### US-SAFE-005: Rate Limiting
- **Приоритет:** MEDIUM | **Статус:** Todo
- **Как** оператор системы, **я хочу** ограничить количество запросов на API, **чтобы** защитить систему от DDoS и abuse.

**Acceptance Criteria:**
- [ ] Auth endpoints: 10 попыток/минута (login), 5 попыток/минута (register)
- [ ] Chat endpoints: 20 сообщений/минута
- [ ] Diagnostics endpoints: 30 запросов/минута
- [ ] При превышении — HTTP 429 Too Many Requests
- [ ] Rate limit по IP + по user_id (если авторизован)

## Эпик 8: PWA

### US-PWA-001: Установка приложения
- **Приоритет:** MEDIUM | **Статус:** Todo
- **Как** пациент, **я хочу** установить Aibolit AI на домашний экран телефона, **чтобы** быстро открывать приложение без ввода URL.

**Acceptance Criteria:**
- [ ] manifest.json: name, short_name, start_url, display: standalone, theme_color, background_color
- [ ] Иконки: 192x192 и 512x512
- [ ] Браузер предлагает «Добавить на экран» (Chrome Android, Safari iOS)
- [ ] Splash screen при загрузке
- [ ] Lighthouse PWA > 90

### US-PWA-002: Offline-справочники
- **Приоритет:** LOW | **Статус:** Todo
- **Как** пациент без интернета, **я хочу** просматривать список специализаций и справочник лабораторных тестов, **чтобы** подготовиться к консультации.

**Acceptance Criteria:**
- [ ] Service worker кэширует: список специализаций, справочник лаб. тестов, статические страницы
- [ ] При отсутствии интернета — offline-страница с доступными справочниками
- [ ] При возврате онлайн — автоматическая синхронизация

### US-PWA-003: Мобильная навигация
- **Приоритет:** MEDIUM | **Статус:** Todo
- **Как** пациент на мобильном устройстве, **я хочу** удобную навигацию, **чтобы** быстро переключаться между разделами.

**Acceptance Criteria:**
- [ ] Bottom navigation bar на мобильных (< 768px): Главная, Чат, Здоровье, Профиль
- [ ] Hamburger menu для дополнительных разделов
- [ ] Touch-friendly элементы: минимум 44x44px
- [ ] Swipe-жесты (назад, обновить)
