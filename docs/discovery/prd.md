# Aibolit AI — PRD (Product Requirements Document)

## 1. Обзор продукта

**Aibolit AI** — бесплатный некоммерческий веб-портал пациента с AI-врачами 35 медицинских специализаций. Портал предоставляет информационные медицинские консультации на основе доказательной медицины для русскоязычных пациентов.

### Ключевые характеристики

| Параметр | Значение |
|----------|----------|
| Тип продукта | WebApp (PWA) |
| Целевой рынок | Россия |
| Монетизация | Нет (бесплатный, некоммерческий) |
| AI-движок | Claude CLI с SSE streaming |
| Специализации | 35 медицинских направлений |
| Статус | MVP функционален |

### Текущий tech stack

- **Frontend:** React 18 + Vite + Tailwind CSS v3 + TypeScript
- **Backend:** FastAPI (Python 3.12) + Uvicorn
- **Database:** SQLite (WAL mode)
- **AI:** Claude CLI с `--session-id` для мультитерновых чатов
- **Auth:** JWT (PyJWT) + SHA-256
- **Интеграции:** PubMed, OpenFDA, RxNorm, WHO ICD-11, ClinicalTrials.gov
- **Инфраструктура:** Docker (backend + nginx)

## 3. Функциональные требования

### 3.1. Существующий функционал (MVP)

#### FR-001: Регистрация и авторизация пациента
- **Статус:** Реализовано
- **Описание:** JWT-аутентификация с регистрацией (3-шаговая форма: учётные данные, персональные данные, медицинская информация). Login/Register на одной странице с табами.
- **Endpoint:** `POST /auth/register`, `POST /auth/login`, `GET /auth/me`

#### FR-002: Профиль пациента
- **Статус:** Реализовано
- **Описание:** CRUD пациента с витальными показателями, лабораторными результатами, диагнозами, назначениями. Dashboard с обзором состояния.
- **Endpoint:** `GET/PUT /patients/{id}`, `GET/POST /patients/{id}/vitals`, `GET/POST /patients/{id}/labs`, и т.д.

#### FR-003: Мультитерновый AI-чат с врачом
- **Статус:** Реализовано
- **Описание:** SSE-стриминг через Claude CLI. Поддержка `--session-id` для хранения контекста диалога. Выбор специализации врача. Отображение дисклеймеров.
- **Endpoint:** `POST /chat/send` (SSE), `GET /chat/history/{session_id}`

#### FR-004: AI-консультации
- **Статус:** Реализовано
- **Описание:** Структурированные консультации с движком клинического рассуждения (doctor.py). 35 специализаций с навыками (skills), связанными ICD-кодами и лабораторными тестами.
- **Endpoint:** `POST /consultations/start`, `GET /consultations/{id}`

#### FR-005: Диагностические инструменты
- **Статус:** Реализовано
- **Описание:** Расшифровка лабораторных анализов, оценка витальных показателей, расчёт СКФ (CKD-EPI), кардиоваскулярный риск (Framingham-like).
- **Endpoint:** `POST /diagnostics/analyze-labs`, `POST /diagnostics/assess-vitals`, `POST /diagnostics/calculate-gfr`, `POST /diagnostics/cardiovascular-risk`

#### FR-006: Лекарственные инструменты
- **Статус:** Реализовано
- **Описание:** Поиск препаратов через OpenFDA, проверка взаимодействий через RxNorm.
- **Endpoint:** `GET /drugs/search`, `POST /drugs/interactions`

#### FR-007: Медицинские документы
- **Статус:** Реализовано
- **Описание:** Генерация и хранение медицинских документов (направления, заключения).
- **Endpoint:** `GET/POST /documents/`

#### FR-008: База знаний
- **Статус:** Реализовано
- **Описание:** Поиск по PubMed, WHO ICD-11, ClinicalTrials.gov.
- **Endpoint:** `GET /knowledge/pubmed/search`, `GET /knowledge/icd/search`, `GET /knowledge/trials/search`

#### FR-009: Справочники
- **Статус:** Реализовано
- **Описание:** Список специализаций, лабораторных тестов с референсными значениями.
- **Endpoint:** `GET /reference/specializations`, `GET /reference/lab-tests`

#### FR-010: Система безопасности
- **Статус:** Частично реализовано
- **Описание:** Детекция red flags (regex-паттерны по категориям: кардио, неврология, психиатрия и т.д.), шаблоны дисклеймеров (8 типов).
- **Модули:** `src/safety/red_flags.py`, `src/safety/disclaimers.py`

#### FR-011: История консультаций
- **Статус:** Реализовано
- **Описание:** Просмотр истории чатов с AI-врачами, история консультаций.
- **Страницы:** ChatHistory, Consultations

### 3.2. Планируемый функционал

#### FR-012: Шифрование медицинских данных at rest
- **Приоритет:** CRITICAL
- **Описание:** Все медицинские данные (диагнозы, результаты анализов, лекарства, аллергии) шифруются в БД. Реализация через column-level encryption для специальной категории ПД (152-ФЗ).
- **Acceptance Criteria:**
  - Все поля из списка `medical_masked_fields` (rules/05) зашифрованы
  - Ключ шифрования хранится в переменной окружения
  - Расшифровка только при чтении авторизованным пользователем
  - Миграция существующих данных в зашифрованный формат

#### FR-013: AuditLogService — аудит-логирование
- **Приоритет:** HIGH
- **Описание:** Структурированное логирование всех значимых событий: CRUD-операции, медицинские события, эскалации, scheduled-задачи. Согласно rules/04-logging.md.
- **Acceptance Criteria:**
  - Модель AuditLog в БД с полями: timestamp, level, action, category, entity_type, entity_id, actor_type, actor_id, message, data, request_id
  - Автоматическое маскирование защищённых полей
  - Категории: db_operation, medical_event, security_event, business_event, task_event
  - Медицинские логи с retention 5 лет
  - Логирование в файлы: app.log, audit.log, medical.log, errors.log
  - Ротация: 10 MB, 5 backup files

#### FR-014: Тесты red flags
- **Приоритет:** HIGH
- **Описание:** Автоматические тесты для RedFlagDetector. Покрытие всех категорий (cardiac, neurological, psychiatric, abdominal, obstetric, allergic, endocrine, vascular, urological, oncological, surgical). False negative rate < 5%.
- **Acceptance Criteria:**
  - Unit-тесты для каждой категории red flags
  - Тесты на false positive (не должны срабатывать на безопасных текстах)
  - Тесты на вариации формулировок (разные способы описать одно состояние)
  - Тесты на edge cases (составные жалобы, аббревиатуры)
  - Метрика false negative rate вычисляется и < 5%

#### FR-015: Тесты дисклеймеров
- **Приоритет:** HIGH
- **Описание:** Проверка, что каждый AI-ответ содержит дисклеймер, соответствующий типу контента.
- **Acceptance Criteria:**
  - Integration-тесты: ответ AI-чата содержит дисклеймер
  - Тип дисклеймера соответствует содержанию (диагностика, лечение, лекарства, анализы и т.д.)
  - Emergency-дисклеймер при red flags
  - Children-дисклеймер при педиатрических консультациях

#### FR-016: PWA (Progressive Web App)
- **Приоритет:** MEDIUM
- **Описание:** Превращение WebApp в PWA для установки на мобильные устройства.
- **Acceptance Criteria:**
  - `manifest.json` с метаданными приложения
  - Service worker для кэширования статики и offline-справочников
  - Иконки (192x192, 512x512)
  - Splash screen
  - Lighthouse PWA score > 90
  - Установка на Android и iOS (Add to Home Screen)

#### FR-017: Мобильная адаптация
- **Приоритет:** MEDIUM
- **Описание:** Полная мобильная адаптация всех 15 страниц портала.
- **Acceptance Criteria:**
  - Responsive layout для всех страниц на экранах 320px-1920px+
  - Touch-friendly элементы управления (минимум 44x44px)
  - Мобильная навигация (hamburger menu или bottom nav)
  - Корректное отображение чата на мобильных устройствах
  - Тестирование на iOS Safari, Android Chrome

## 5. Ограничения и допущения

### Ограничения

1. **Регуляторные:** AI-система не является медицинским изделием (не проходила регистрацию в Росздравнадзоре). Все ответы — информационного характера.
2. **Технические:** Зависимость от Claude CLI (Anthropic) для AI-ответов. При недоступности API консультации невозможны.
3. **Финансовые:** Бесплатный проект — бюджет ограничен. API-вызовы Claude оплачиваются из собственных средств.
4. **Данные:** SQLite — ограничение по конкурентности записи. Для 100+ одновременных пользователей может потребоваться миграция на PostgreSQL.
5. **Юридические:** Не может выписывать рецепты, больничные, заключения. Не несёт юридической ответственности за медицинские решения пациента.

### Допущения

1. Claude CLI будет доступен и стабилен для production-использования
2. Пользователи принимают, что получают информационные (не медицинские) консультации
3. Пользователи дают информированное согласие на обработку медицинских данных
4. Интернет-покрытие целевой аудитории достаточно для работы с SSE-стримингом
5. Red flags детектор покрывает наиболее критические экстренные состояния
