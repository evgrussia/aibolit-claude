# User Flows и Information Architecture

## 1. Карта навигации (Sitemap)

### Полная структура всех 15 страниц

```
/login                                  LoginPage
  |
  +-- / (redirect)                      RedirectToMyDashboard
  |
  +-- /patients/:patientId              DashboardPage
  |     +-- /labs                       LabResultsPage
  |     +-- /vitals                     VitalsHistoryPage
  |     +-- /consultations              ConsultationsPage
  |     +-- /timeline                   HealthTimelinePage
  |     +-- /diagnoses/:icdCode         DiagnosisDetailPage

  +-- /consult                          ConsultDoctorPage

  |     +-- /chat/:consultationId       ChatPage
  +-- /diagnostics                      DiagnosticsPage
  +-- /drugs                            DrugToolsPage
```

```
LoginPage ──────────────────────> DashboardPage
                                      |
                    +-----------------+-----------------+
                    |                 |                 |
              LabResultsPage  VitalsHistoryPage  HealthTimelinePage
                    |                 |
                    +--------+--------+
                             |
```

## 2. User Flows

### User Flow #1: Регистрация -> Первая консультация

**Персона:** Новый пользователь, 35 лет, хочет получить AI-консультацию по жалобе.

```
[1] Открытие портала
    |
    v
[2] LoginPage: вкладка "Регистрация"
    |
    v
[3] Шаг 1/3: Учётные данные
    - Логин
    - Пароль
    - Подтверждение пароля
    - [Далее]
    |
    v
[4] Шаг 2/3: Личные данные
    - Имя
    - Фамилия
    - Дата рождения
    - Пол
    - [Далее]
    |
    v
[5] Шаг 3/3: Медицинские данные
    - Группа крови (необязательно)
    - Аллергии (необязательно)
    - Семейный анамнез (необязательно)
    - [Зарегистрироваться]
    |
    v
[6] Автоматический вход -> Redirect на DashboardPage
    |
    v
[7] DashboardPage: знакомство с панелью
    - Карточка пациента
    - Виджеты витальных (пусто)
    - Диагнозы (пусто)
    - Назначения (пусто)
    |
    v
[8] Навигация: Sidebar -> "AI Консультация" (/consult)
    |
    v
[9] ConsultDoctorPage: выбор специалиста
    - Поиск по специализации
    - 35 специализаций, 5 категорий
    - Выбор специалиста (например, "Терапевт")
    |
    v
[10] Описание жалобы
     - Текстовое поле жалобы
     - [Начать консультацию]
     |
     v
[11] Triage (маршрутизация)
     - AI анализирует жалобу
     - Рекомендуемые специалисты с релевантностью
     - Red flags (если есть)
     - Выбор специалиста -> [Начать чат]
     |
     v
[12] ChatPage: AI-чат со специалистом
     - SSE streaming ответа
     - Мультитерновый диалог
     - Дисклеймер в каждом ответе
     - Markdown-форматирование
     |
     v
[13] Получение рекомендации
     - AI предоставляет информационный ответ
     - Дисклеймер: "Обратитесь к врачу"
     - Возможность задать уточняющий вопрос
```

**Точки выхода:**
- Шаг 3: если пользователь закрывает страницу -> данные потеряны
- Шаг 9: если пользователь не может выбрать специалиста -> поиск
- Шаг 12: если обнаружен red flag -> экстренное предупреждение (User Flow #5)

### User Flow #2: Обращение с жалобой -> AI-чат -> Получение рекомендации

**Персона:** Существующий пользователь, 50 лет, болит голова уже 3 дня.

```
[1] Авторизация (LoginPage) или уже залогинен
    |
    v
[2] Способ A: Dashboard -> Sidebar -> "Чат с врачом" (/chat)
    Способ B: Dashboard -> Sidebar -> "AI Консультация" (/consult)
    |
    +-- Способ A -------------------------+
    |                                      |
    v                                      v
[3A] ChatHistoryPage                [3B] ConsultDoctorPage
     - Список предыдущих чатов           - Сетка специализаций
     - [Новая консультация]              - Поиск
     |                                    - Выбор "Невролог"
     v                                    |
[4A] -> Redirect на /consult             v
         (выбор специалиста)        [4B] Форма описания жалобы
                                         - "Головная боль 3 дня, усиливается
                                            при наклоне, тошнота"
                                         - [Начать консультацию]
                                         |
                                         v
                                    [5] TriageResult
                                         - Рекомендовано: Невролог (85%)
                                         - Также: Терапевт (60%)
                                         - Red flags: проверка на
                                           менингеальные знаки
                                         - [Выбрать невролога]
                                         |
                                         v
[6] ChatPage (/chat/:consultationId)
    |
    +-- Сообщение пациента: "Болит голова 3 дня..."
    |
    v
[7] AI-невролог анализирует (SSE streaming):
    - Уточняющие вопросы:
      "Какой характер боли? Где локализуется?
       Есть ли светобоязнь? Принимали ли что-то?"
    |
    v
[8] Пациент отвечает на вопросы
    |
    v
[9] AI формирует рекомендацию:
    - Возможные причины (дифференциальный диагноз)
    - Рекомендуемые обследования
    - К какому специалисту обратиться
    - Дисклеймер
    |
    v
[10] Пациент может:
     a) Задать ещё вопрос -> [8]
     b) Завершить чат -> ChatHistoryPage
     c) Перейти в "Диагностика" -> DiagnosticsPage
```

**Альтернативные сценарии:**
- Если AI не может определить специализацию -> предлагает терапевта
- Если жалоба из другой области -> перенаправление к другому специалисту
- Если обнаружены red flags -> User Flow #5

### User Flow #3: Добавление витальных -> Просмотр трендов -> Диагностика

**Персона:** Пациент с гипертензией, 55 лет, ежедневно измеряет давление.

```
[1] DashboardPage
    |
    v
[2] Виджет "Витальные показатели"
    - VitalsGauges с текущими значениями
    - Кнопка [+ Добавить измерение]
    |
    v
[3] Modal: AddVitalsForm
    - АД систолическое: 145
    - АД диастолическое: 92
    - ЧСС: 78
    - Температура: 36.6
    - SpO2: 97
    - ЧД: 16
    - Глюкоза: 5.4
    - [Сохранить]
    |
    v
[4] Toast: "Показатели сохранены"
    - VitalsGauges обновлены
    - Визуальная индикация:
      * Зелёный: SpO2, ЧСС, температура — норма
      * Жёлтый/оранжевый: АД 145/92 — повышено
    |
    v
[5] Навигация: Sidebar -> "Витальные" (/patients/:id/vitals)
    |
    v
[6] VitalsHistoryPage
    - BloodPressureChart: тренд АД за период
    - Таблица измерений
    - Фильтр по периоду
    |
    v
[7] Анализ тренда:
    - Пациент видит, что АД стабильно выше нормы
    - Решает получить оценку AI
    |
    v
[8] Навигация: Sidebar -> "Диагностика" (/diagnostics)
    |
    v
[9] DiagnosticsPage -> Таб "Оценка витальных"
    - VitalsAssessTab
    - Загрузка текущих витальных
    - [Оценить]
    |
    v
[10] Результат AI-оценки:
     - severity: "warning"
     - alerts: ["Систолическое АД выше нормы"]
     - Рекомендация: обратиться к кардиологу
     - MedicalDisclaimer type="general"
     |
     v
[11] Пациент может:
     a) Перейти к кардиориску -> Таб "Кардиориск" (CvRiskTab)
     b) Начать консультацию с кардиологом -> /consult
     c) Посмотреть анализы -> Таб "Анализы" (LabAnalysisTab)
```

### User Flow #4: Поиск лекарства -> Проверка взаимодействий

**Персона:** Пациент, 60 лет, принимает несколько препаратов, хочет проверить совместимость.

```
[1] DashboardPage или любая страница
    |
    v
[2] Sidebar -> "Лекарства" (/drugs)
    |
    v
[3] DrugToolsPage: две секции
    |
    +-- Секция 1: DrugSearchSection ----+
    |                                    |
    v                                    |
[4] Поиск лекарства                     |
    - Ввод: "Амлодипин"                 |
    - Автодополнение                     |
    - Результат: описание, показания,   |
      противопоказания, побочные        |
    - MedicalDisclaimer type="medication"|
    |                                    |
    +------------------------------------+
    |
    +-- Секция 2: InteractionSection ----+
    |                                    |
    v                                    |
[5] Проверка взаимодействий              |
    - Добавить препарат 1: "Амлодипин"  |
    - Добавить препарат 2: "Варфарин"   |
    - [Проверить взаимодействия]         |
    |                                    |
    v                                    |
[6] Результат проверки:                  |
    - severity: "major" / "moderate" /   |
      "minor"                            |
    - Описание взаимодействия            |
    - Рекомендации                       |
```

### User Flow #5: Red Flag -> Экстренное предупреждение -> Рекомендация вызвать 103

**Персона:** Любой пациент, описавший экстренные симптомы.

```
[1] ChatPage: AI-чат с любым специалистом
    |
    v
[2] Пациент вводит сообщение:
    "Внезапно сильно заболела голова, как удар,
     онемела правая рука, трудно говорить"
    |
    v
[3] AI-система детектирует RED FLAGS:
    - "Громоподобная" головная боль
    - Онемение одной стороны тела (инсульт)
    - Нарушение речи (инсульт)
    |
    v
[4] НЕМЕДЛЕННАЯ РЕАКЦИЯ:
    |
    +-- EmergencyBanner (role="alert"):
    |   +-----------------------------------------+
    |   | [Phone] ВНИМАНИЕ! Обнаружены критические |
    |   | показатели, требующие экстренной         |
    |   | медицинской помощи!                      |
    |   |                                          |
    |   | [103]  [112]                             |
    |   | Немедленно вызовите скорую помощь!       |
    |   +-----------------------------------------+
    |
    +-- AI-ответ в чате:
    |   "Описанные вами симптомы МОГУТ УКАЗЫВАТЬ
    |    на острое нарушение мозгового
    |    кровообращения (инсульт).
    |
    |    НЕМЕДЛЕННО вызовите скорую помощь: 103
    |
    |    До приезда скорой:
    |    - Не принимайте лекарства самостоятельно
    |    - Лягте, приподняв голову
    |    - Не ешьте и не пейте
    |    - Запомните время начала симптомов"
    |
    +-- Дисклеймер emergency:
        "Информация предоставлена AI. При угрозе
         жизни немедленно вызовите скорую: 103/112"
    |
    v
[5] Действия системы (фоново):
    - Логирование: red_flag_detected
    - Логирование: doctor_escalation_triggered
    - Уведомление дежурного врача (если доступно)
    |
    v
[6] Пациент может:
    a) Нажать кнопку [103] -> звонок
    b) Нажать кнопку [112] -> звонок
    c) Продолжить чат -> AI продолжает рекомендовать
       вызвать скорую, не отвлекая разговором
```

**Триггеры Red Flag (из 05-medical-safety.md):**

| Категория | Примеры |
|-----------|---------|
| Кардиологические | Острая боль в груди, одышка в покое, потеря сознания |
| Неврологические | Слабость одной стороны, потеря зрения, громоподобная головная боль |
| Аллергические | Анафилаксия, отёк Квинке |
| Травматологические | Обильное кровотечение, травма позвоночника |
| Психиатрические | Суицидальные мысли, острый психоз |
| Педиатрические | Лихорадка >39 C у ребёнка до 3 мес., отказ от еды |
| Акушерские | Кровотечение при беременности |
| Абдоминальные | Острый живот, рвота кровью, мелена |
| Инфекционные | Лихорадка >40 C, менингеальные знаки |

## 3. Information Architecture

### Группировка страниц по разделам

```yaml
Мои данные (Patient-scoped):
  description: "Персональные медицинские данные пациента"
  pages:
    - DashboardPage:       "Обзорная панель — всё в одном месте"
    - HealthTimelinePage:  "Хронологический вид событий"
    - LabResultsPage:      "Анализы с трендами и расшифровкой"
    - VitalsHistoryPage:   "История витальных показателей"
    - DiagnosisDetailPage: "Детали конкретного диагноза (МКБ-10)"
    - ConsultationsPage:   "Список консультаций пациента"

Инструменты (Global):
  description: "AI-инструменты и утилиты для здоровья"
  pages:
    - ChatHistoryPage:     "История всех чатов с AI-врачами"
    - ChatPage:            "Мультитерновый чат с AI-врачом"
    - ConsultDoctorPage:   "Выбор специалиста и начало консультации"
    - DiagnosticsPage:     "Диагностические инструменты (анализы, витальные, СКФ, кардиориск)"
    - DrugToolsPage:       "Поиск лекарств и проверка взаимодействий"
    - DocumentsPage:       "Документы, рецепты, направления"

Системные:
  description: "Аутентификация и настройки"
  pages:
    - LoginPage:           "Вход и регистрация"
    - SettingsPage:        "Профиль и настройки приложения"
```

### Иерархия контента по приоритету

```
Уровень 1 — Критический (всегда доступно):
  - EmergencyBanner (экстренные ситуации)
  - MedicalDisclaimer (в каждом AI-ответе)
  - Кнопки 103/112

Уровень 2 — Первичный (главные действия):
  - Dashboard: обзор здоровья
  - Чат с врачом: основная функция
  - AI Консультация: начало новой консультации

Уровень 3 — Вторичный (детализация):
  - Анализы, витальные, хронология
  - Диагностика (табы)
  - Лекарства
  - Документы

Уровень 4 — Вспомогательный:
  - Настройки
  - Breadcrumbs (навигационные хлебные крошки)
  - Help Tooltips
```

## 4. Навигационная модель (Sidebar)

### Текущая структура Sidebar

```
+------------------------------------------+
|  [Heart] Aibolit                         |
|  [User] username                         |
+------------------------------------------+
|                                          |
|  МОИ ДАННЫЕ                              |
|  [LayoutDashboard] Панель                |
|  [Clock]           Хронология            |
|  [FlaskConical]    Анализы               |
|  [HeartPulse]      Витальные             |
|                                          |
|  ИНСТРУМЕНТЫ                             |
|  [MessageCircle]   Чат с врачом          |
|  [MessageSquare]   AI Консультация       |
|  [Activity]        Диагностика           |
|  [Pill]            Лекарства             |
|  [FileText]        Документы             |
|                                          |
+------------------------------------------+
|  35 специализаций, 31 инструмент         |
|  [!] Только для демо и обучения          |
|  [Settings] Настройки                    |
|  [LogOut] Выйти                          |
+------------------------------------------+
```

### Принципы навигации

1. **Группировка:** две секции — "Мои данные" (patient-scoped) и "Инструменты" (global)
2. **Mobile-first:** на мобильных sidebar скрыт, открывается через hamburger-меню
3. **Активный элемент:** подсветка текущей страницы (bg-white/20, text-white)
4. **Иконки:** Lucide React, размер 18px, согласованный стиль
5. **Группы:** заголовки uppercase, мелкий шрифт, цвет blue-300
6. **Футер:** метаданные, предупреждение о демо, настройки, выход

### Навигационные паттерны

| Паттерн | Описание | Пример |
|---------|----------|--------|
| Sidebar NavLink | Основная навигация | Все пункты меню |
| Breadcrumbs | Контекстная навигация | `/patients/7facb1f2/labs` -> "ID 7facb1f2 > Анализы" |
| In-page tabs | Переключение внутри страницы | DiagnosticsPage: 4 таба |
| Card-as-link | Карточка как точка входа | Клик по диагнозу -> DiagnosisDetailPage |
| Button redirect | Действие -> новая страница | "Начать консультацию" -> ChatPage |
| Modal | Действие без ухода со страницы | AddVitalsForm, AddLabResultForm |
