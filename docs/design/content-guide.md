# Content Guide / UX Copy — Aibolit AI

> Руководство по контенту, тону коммуникации и формулировкам для медицинского портала Aibolit AI

---

## 1. Tone of Voice

### Основные характеристики

| Характеристика | Описание | Пример |
|----------------|----------|--------|
| **Уважительный** | Обращение на "вы", без покровительственного тона | "Вы можете обратиться к специалисту" (не "Тебе надо к врачу") |
| **Понятный** | Медицинские термины с пояснениями | "Гемоглобин (белок, переносящий кислород в крови)" |
| **Не пугающий** | Спокойное изложение без нагнетания | "Рекомендуется обследование" (не "Срочно! Это может быть опасно!") |
| **Честный** | Прямое указание ограничений AI | "AI-система не заменяет врача" |
| **Заботливый** | Проявление внимания к пациенту | "Мы понимаем ваше беспокойство" |
| **Профессиональный** | Корректная медицинская терминология | Использование МКБ-10, общепринятых аббревиатур |

### Что делать и чего избегать

| Делать | Избегать |
|--------|----------|
| Использовать "вы" (с маленькой буквы в интерфейсе) | Тыкать, использовать фамильярный тон |
| Давать контекст медицинским терминам | Оставлять термины без объяснений |
| Предлагать действия | Ставить диагнозы |
| Указывать на ограничения | Создавать иллюзию всезнания |
| Быть кратким в интерфейсе | Перегружать текстом |
| Использовать нейтральный язык | Использовать эмоциональные формулировки |
| Проявлять эмпатию | Быть безразличным или роботизированным |

### Градация серьёзности в тоне

| Контекст | Тон | Пример |
|----------|-----|--------|
| Обычный интерфейс | Нейтрально-дружелюбный | "Добавить результат анализа" |
| AI-рекомендация | Профессионально-осторожный | "На основе описанных симптомов, возможными причинами могут быть..." |
| Предупреждение | Серьёзный, но не пугающий | "Обратите внимание: показатель выше нормы" |
| Red Flag / Emergency | Прямой и однозначный | "ВНИМАНИЕ! Немедленно вызовите скорую помощь: 103" |

## 2. Правила формулировок для AI-ответов

### Допустимые формулировки

Следующие конструкции РАЗРЕШЕНЫ в ответах AI-системы:

```
Анализ симптомов:
  "На основе описанных симптомов, ВОЗМОЖНЫМИ причинами могут быть..."
  "В медицинской литературе при таких симптомах РАССМАТРИВАЮТСЯ..."
  "Описанные проявления МОГУТ БЫТЬ СВЯЗАНЫ с..."
  "По данным клинических рекомендаций, ОДНОЙ ИЗ ВОЗМОЖНЫХ причин является..."

Рекомендации:
  "РЕКОМЕНДУЕТСЯ обратиться к [специалист] для уточнения..."
  "ДЛЯ УТОЧНЕНИЯ диагноза может быть полезно пройти..."
  "РАССМОТРИТЕ возможность консультации с..."
  "Врач МОЖЕТ РЕКОМЕНДОВАТЬ дополнительное обследование..."

Лекарства:
  "При подобных состояниях в практике МОЖЕТ РАССМАТРИВАТЬСЯ [препарат]"
  "Согласно клиническим рекомендациям, при таких показателях МОЖЕТ ПРИМЕНЯТЬСЯ..."
  "Консультация врача перед приёмом любых препаратов ОБЯЗАТЕЛЬНА"

Анализы:
  "Данный показатель НАХОДИТСЯ ВНЕ РЕФЕРЕНСНОГО ДИАПАЗОНА..."
  "Отклонение от нормы НЕ ОБЯЗАТЕЛЬНО означает заболевание..."
  "Для корректной интерпретации НЕОБХОДИМА консультация врача..."

Общее:
  "Следует отметить, что..."
  "Важно учитывать, что..."
  "Необходимо подчеркнуть, что AI-система не заменяет врача..."
```

### Запрещённые формулировки

Следующие конструкции ЗАПРЕЩЕНЫ (категорически):

```
Диагнозы:
  "У вас [диагноз]"
  "Это [заболевание]"
  "Это точно / несомненно / определённо..."
  "Могу с уверенностью сказать..."
  "Диагноз: [название]"

Назначения:
  "Вам нужно принимать [препарат]"
  "Принимайте [препарат] по [схема]"
  "Я назначаю вам..."
  "Рекомендую начать приём..."
  "Прекратите приём [препарат]"

Исключения опасных состояний:
  "Это не опасно"
  "Беспокоиться не о чем"
  "Ничего страшного"
  "Можете не переживать"

Юридическое:
  "Как ваш врач, я рекомендую..."
  "Мой диагноз..."
  "Назначаю обследование..."
```

### Таблица замены: неправильно -> правильно

| Неправильно | Правильно |
|-------------|-----------|
| "У вас гастрит" | "Симптомы МОГУТ указывать на гастрит" |
| "Принимайте ибупрофен 400мг" | "При подобных состояниях МОЖЕТ рассматриваться ибупрофен. Дозировку определяет врач" |
| "Это не опасно" | "Для исключения серьёзных состояний рекомендуется обратиться к врачу" |
| "Сдайте анализ крови" | "Для уточнения может быть полезен общий анализ крови" |
| "Вам нужен кардиолог" | "Рекомендуется консультация кардиолога" |
| "Анализы в норме" | "Представленные показатели находятся в пределах референсного диапазона" |
| "Ваш диагноз подтверждён" | "Для подтверждения предполагаемого диагноза рекомендуется обратиться к специалисту" |

## 3. Шаблоны дисклеймеров

### Полный перечень дисклеймеров

Каждый дисклеймер отображается компонентом `MedicalDisclaimer` (`bg-amber-50`, `border-amber-200`, иконка `AlertTriangle`).

#### 3.1. Общий (general)

**Когда:** Каждый ответ AI-агента.

> Информация предоставлена AI-ассистентом и носит исключительно информационный характер. Не является медицинской консультацией, диагнозом или назначением лечения. Обратитесь к врачу для получения квалифицированной медицинской помощи.

#### 3.2. Диагностический (diagnosis)

**Когда:** При анализе симптомов или предложении возможных диагнозов.

> Предположительный анализ симптомов выполнен AI-системой. Это НЕ диагноз. Только врач может поставить диагноз на основе полного обследования. Обратитесь к специалисту.

#### 3.3. Лечебный (treatment)

**Когда:** При упоминании лечения, терапии, процедур.

> Информация о лечении предоставлена в образовательных целях. НЕ начинайте, не изменяйте и не прекращайте лечение без консультации с врачом. Самолечение может быть опасным.

#### 3.4. Лекарственный (medication)

**Когда:** При упоминании конкретных лекарств, дозировок.

> Информация о лекарственных препаратах носит справочный характер. AI НЕ назначает лекарства. Перед приёмом любых препаратов проконсультируйтесь с врачом. Учитывайте противопоказания и индивидуальную непереносимость.

#### 3.5. Лабораторный (lab_analysis)

**Когда:** При интерпретации лабораторных анализов.

> Расшифровка анализов выполнена AI-системой и требует подтверждения врачом. Отклонения от нормы не обязательно означают заболевание. Врач интерпретирует результаты в контексте вашего состояния.

#### 3.6. Визуализация (imaging)

**Когда:** При анализе медицинских изображений (рентген, КТ, дерматоскопия).

> Анализ изображения выполнен AI-моделью и является предварительным скринингом. Это НЕ радиологическое заключение. Окончательную интерпретацию должен выполнить врач-рентгенолог.

#### 3.7. Экстренный (emergency)

**Когда:** При обнаружении признаков экстренного состояния. Отображается компонентом `EmergencyBanner` (`bg-red-50`, `border-2 border-red-300`, `role="alert"`).

> ВНИМАНИЕ! При угрозе жизни немедленно вызовите скорую помощь: 103 (или 112). AI-система НЕ предназначена для экстренных ситуаций.

#### 3.8. Педиатрический (children)

**Когда:** При консультации по педиатрии.

> Информация о здоровье детей требует особой осторожности. Всегда консультируйтесь с педиатром. Дозировки и рекомендации для взрослых НЕ применимы к детям.

### Правила размещения дисклеймеров

1. Дисклеймер размещается **В КОНЦЕ** каждого ответа AI
2. Тип дисклеймера **соответствует содержанию** ответа
3. При нескольких типах контента используются **ВСЕ** релевантные дисклеймеры
4. Emergency дисклеймер (EmergencyBanner) показывается **ПЕРВЫМ**, если применим
5. Дисклеймер **нельзя скрыть, свернуть или уменьшить**

## 4. Терминология: медицинские термины + простые пояснения

### Принцип Health Literacy

Каждый медицинский термин при первом упоминании сопровождается пояснением в скобках или через тире. Повторные упоминания могут использовать только термин.

### Словарь основных терминов

| Медицинский термин | Простое пояснение | Контекст использования |
|-------------------|-------------------|----------------------|
| Артериальное давление (АД) | Давление крови в сосудах | Витальные показатели |
| Систолическое давление | Верхнее давление (в момент сокращения сердца) | Витальные, VitalsGauges |
| Диастолическое давление | Нижнее давление (в момент расслабления сердца) | Витальные, VitalsGauges |
| ЧСС | Частота сердечных сокращений (пульс) | Витальные |
| SpO2 | Насыщение крови кислородом | Витальные |
| ЧД | Частота дыхания (вдохов в минуту) | Витальные |
| Гемоглобин | Белок крови, переносящий кислород | Анализы |
| Глюкоза натощак | Уровень сахара в крови до еды | Анализы |
| АЛТ / АСТ | Ферменты печени (показатели работы печени) | Анализы |
| Креатинин | Показатель работы почек | Анализы, СКФ |
| СКФ (GFR) | Скорость клубочковой фильтрации (показывает, как хорошо работают почки) | Калькулятор СКФ |
| ХБП (CKD) | Хроническая болезнь почек | Калькулятор СКФ |
| HbA1c | Гликированный гемоглобин (средний сахар за 3 месяца) | Анализы |
| ТТГ | Тиреотропный гормон (показатель работы щитовидной железы) | Анализы |
| МКБ-10 | Международная классификация болезней | Диагнозы |
| Холестерин | Жироподобное вещество в крови | Анализы, кардиориск |
| Тахикардия | Учащённое сердцебиение (пульс > 100) | Диагностика |
| Брадикардия | Замедленное сердцебиение (пульс < 60) | Диагностика |
| Гипертензия | Повышенное артериальное давление | Диагностика |
| Гипотензия | Пониженное артериальное давление | Диагностика |
| Анафилаксия | Тяжёлая аллергическая реакция, угрожающая жизни | Red flags |
| Менингеальные знаки | Признаки воспаления оболочек мозга | Red flags |
| Мелена | Чёрный дёгтеобразный стул (признак кровотечения) | Red flags |
| Дифференциальный диагноз | Список возможных заболеваний при данных симптомах | AI-ответ |
| Референсный диапазон | Границы нормы для анализа | Анализы |

### Правила использования терминов в интерфейсе

1. **Названия страниц и кнопок** — используются короткие понятные формулировки: "Анализы", "Витальные", "Диагностика"
2. **Результаты AI** — термин + пояснение при первом упоминании: "Креатинин (показатель работы почек)"
3. **Таблицы данных** — используются медицинские аббревиатуры с подсказками через `HelpTooltip`: "АЛТ [?]"
4. **Нормы** — всегда указывать единицы измерения: "120-150 г/л", "3.9-6.1 ммоль/л"

## 6. Формулировки Red Flags и экстренных ситуаций

### Шаблон экстренного сообщения

При обнаружении red flag AI обязан использовать следующий шаблон:

```
ВНИМАНИЕ: Обнаружены признаки, требующие срочной медицинской помощи!

Что обнаружено: [описание red flag простым языком]

Рекомендация: Немедленно вызовите скорую помощь (103 или 112)

До приезда помощи: [краткие безопасные действия, если применимо]
```

### Формулировки по категориям

#### Кардиологические

```
Red Flag: Острая боль в груди
Сообщение: "Описанные симптомы (боль в груди, отдающая в левую руку)
МОГУТ УКАЗЫВАТЬ на острый коронарный синдром. Это состояние
ТРЕБУЕТ НЕМЕДЛЕННОЙ медицинской помощи. Вызовите скорую: 103."

До приезда: "Примите полусидячее положение. Не делайте резких движений.
Если есть нитроглицерин, назначенный врачом, примите его."
```

#### Неврологические

```
Red Flag: Признаки инсульта
Сообщение: "Описанные симптомы (онемение одной стороны тела,
нарушение речи) МОГУТ УКАЗЫВАТЬ на острое нарушение мозгового
кровообращения. НЕМЕДЛЕННО вызовите скорую: 103."

До приезда: "Уложите человека, приподняв голову. Не давайте еду
и питьё. Запомните время начала симптомов — это важно для врачей."
```

#### Аллергические

```
Red Flag: Анафилаксия
Сообщение: "Описанные симптомы (отёк горла, затруднённое дыхание,
сыпь) МОГУТ УКАЗЫВАТЬ на анафилактическую реакцию. Это состояние
УГРОЖАЕТ ЖИЗНИ. НЕМЕДЛЕННО вызовите скорую: 103."

До приезда: "Если есть автоинъектор адреналина, назначенный врачом,
используйте его. Примите полусидячее положение."
```

#### Психиатрические

```
Red Flag: Суицидальные мысли
Сообщение: "Мы понимаем, что вам сейчас тяжело. Ваша жизнь важна.
Пожалуйста, обратитесь за помощью прямо сейчас:
- Телефон доверия: 8-800-2000-122 (бесплатно, круглосуточно)
- Скорая помощь: 103
Специалисты помогут вам."
```

### Правила экстренных формулировок

1. **Не задерживать** пациента разговором — сразу предупредить
2. **Не паниковать** — чёткие, спокойные инструкции
3. **Давать конкретные номера** телефонов (103, 112, 8-800-2000-122)
4. **Только безопасные действия** до приезда скорой (ничего инвазивного)
5. **Не диагностировать** — использовать "МОГУТ УКАЗЫВАТЬ", "ВОЗМОЖНО"
6. **Приоритет** — безопасность пациента выше продолжения консультации

## 7. Уровни уверенности AI: отображение

### Визуальная система уровней

| Уровень | Процент | Отображаемый текст | Цвет | Badge variant |
|---------|---------|---------------------|------|---------------|
| Высокая | >80% | "Высокая вероятность" | emerald-600 | success |
| Умеренная | 50-80% | "Умеренная вероятность" | amber-600 | warning |
| Низкая | 20-50% | "Возможная причина" | gray-500 | gray |
| Очень низкая | <20% | "Маловероятно, но не исключено" | gray-400 | gray |

### Шаблоны текста для уровней

#### Высокая вероятность (>80%)

```
"На основе описанных симптомов, наиболее ВЕРОЯТНОЙ причиной
может быть [состояние]. Рекомендуется обратиться к [специалист]
для подтверждения и назначения лечения."

Сопутствующий дисклеймер: "Требуется подтверждение врачом"
```

#### Умеренная вероятность (50-80%)

```
"Описанные симптомы МОГУТ БЫТЬ СВЯЗАНЫ с [состояние].
Для уточнения рекомендуется пройти [обследование].
Также следует рассмотреть [альтернативные причины]."

Сопутствующий дисклеймер: "Необходимо дополнительное обследование"
```

#### Низкая вероятность (20-50%)

```
"Среди возможных причин РАССМАТРИВАЕТСЯ [состояние],
однако для подтверждения необходимо комплексное обследование.
Другие возможные причины: [список]."

Сопутствующий дисклеймер: "Одна из возможных причин, требуется обследование"
```

#### Очень низкая вероятность (<20%)

```
"[Состояние] МАЛОВЕРОЯТНО, но не может быть полностью исключено.
Для полного исключения рекомендуется [обследование]."

Сопутствующий дисклеймер: "Маловероятно, но для исключения
рекомендуется обследование"
```

### Правила отображения уровней уверенности

1. **Всегда указывать уровень** для каждого предположения AI
2. **Минимум 2-3 альтернативы** при дифференциальном диагнозе
3. **Не показывать числовой процент** пациенту напрямую (кроме экрана TriageResult для релевантности специалиста) — использовать текстовые метки
4. **Сортировка** от наиболее к наименее вероятному
5. **При очень низкой уверенности** — упоминать как дополнительный вариант, не акцентировать
6. **Ни один уровень не исключает** необходимости обращения к врачу
