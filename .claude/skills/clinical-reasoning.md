## Pipeline клинического рассуждения

### Этапы

```yaml
1. Symptom Analysis (Анализ симптомов):
   - Извлечение симптомов из текста пациента (NER)
   - Классификация: основные / сопутствующие
   - Оценка severity (1-10)
   - Определение onset, duration, character
   - Маппинг на SNOMED CT коды

2. Red Flag Screening (Скрининг красных флагов):
   - Проверка по базе red flags (rules/05-medical-safety.md)
   - Если обнаружен → немедленная эскалация (urgency 4-5)
   - Логирование: red_flag_detected

3. History Integration (Интеграция анамнеза):
   - Текущие жалобы + анамнез заболевания
   - Хронические заболевания
   - Аллергии и непереносимости
   - Текущие лекарства
   - Семейный анамнез (при наличии)

4. Differential Diagnosis (Дифференциальная диагностика):
   - Генерация гипотез на основе симптомов
   - Ранжирование по вероятности (Bayesian reasoning)
   - Минимум 2-3 дифференциальных диагноза
   - Для каждого: МКБ-10 код, confidence level, evidence grade

5. Evidence Grading (Грейдирование доказательств):
   - GRADE система (A/B/C/D/E)
   - Ссылки на источники
   - Уровень уверенности AI

6. Recommendation Formation (Формирование рекомендаций):
   - Рекомендуемые обследования
   - К какому специалисту обратиться
   - Информационные рекомендации (НЕ назначения)
   - Обязательный дисклеймер
```

## Структуры данных

### SymptomAnalysis

```python
from dataclasses import dataclass, field
from enum import Enum
from typing import Optional

class Severity(Enum):
    MILD = "mild"           # 1-3
    MODERATE = "moderate"   # 4-6
    SEVERE = "severe"       # 7-9
    CRITICAL = "critical"   # 10

@dataclass
class Symptom:
    name: str                          # Название симптома
    snomed_code: Optional[str] = None  # SNOMED CT код
    severity: int = 5                  # 1-10
    onset: Optional[str] = None        # Когда началось
    duration: Optional[str] = None     # Длительность
    character: Optional[str] = None    # Характер (острая, тупая, ноющая)
    location: Optional[str] = None     # Локализация
    aggravating: list[str] = field(default_factory=list)  # Усиливающие факторы
    relieving: list[str] = field(default_factory=list)    # Облегчающие факторы

@dataclass
class DifferentialDiagnosis:
    icd10_code: str                    # МКБ-10 код
    name: str                          # Название диагноза
    confidence: float                  # 0.0-1.0
    evidence_grade: str                # A/B/C/D/E
    supporting_symptoms: list[str]     # Подтверждающие симптомы
    contradicting_symptoms: list[str]  # Противоречащие симптомы
    recommended_tests: list[str]       # Рекомендуемые обследования
    specialist: Optional[str] = None   # Рекомендуемый специалист

@dataclass
class ClinicalReasoning:
    symptoms: list[Symptom]
    red_flags: list[str]
    differential: list[DifferentialDiagnosis]
    recommended_actions: list[str]
    disclaimer_type: str  # general/diagnosis/treatment/emergency
    escalation_needed: bool = False
    escalation_urgency: int = 0  # 0-5
```

## Алгоритм дифференциальной диагностики

### Bayesian Reasoning

```yaml
Принцип:
  P(Disease|Symptoms) ∝ P(Symptoms|Disease) × P(Disease)

Шаги:
  1. Prior probability — базовая распространённость заболевания
     (по эпидемиологическим данным для региона, возраста, пола)

  2. Likelihood — вероятность набора симптомов при данном заболевании
     (из клинических рекомендаций и RAG)

  3. Posterior — итоговая вероятность
     (нормализованная по всем гипотезам)

  4. Ranking — ранжирование гипотез по posterior probability

Ограничения:
  - AI работает с ПРИБЛИЖЁННЫМИ вероятностями
  - Не учитывает все факторы (отсутствие физикального осмотра)
  - Confidence level отражает НЕОПРЕДЕЛЁННОСТЬ, а не точность диагноза
```

### Pattern Matching

```yaml
Дополнительно к Bayesian:
  - Классические паттерны заболеваний (symptom clusters)
  - Возрастные и половые особенности
  - Сезонные факторы
  - Эндемичность региона

Примеры паттернов:
  chest_pain_cardiac:
    symptoms: [давящая боль за грудиной, иррадиация в левую руку, одышка]
    age_factor: ">40 увеличивает вероятность"
    risk_factors: [курение, диабет, гипертензия, семейный анамнез]

  appendicitis:
    symptoms: [боль в правой подвздошной области, тошнота, субфебрильная температура]
    classic_sequence: "боль начинается в эпигастрии, мигрирует вправо"
    red_flag: true
```

## Evidence Grading (GRADE)

```yaml
Уровни доказательности:

  A — Высокий:
    sources: "Систематические обзоры РКИ, мета-анализы"
    certainty: "Высокая уверенность в эффекте"
    ai_action: "Можно формулировать как 'рекомендуется'"

  B — Умеренный:
    sources: "Отдельные РКИ, крупные когортные исследования"
    certainty: "Умеренная уверенность"
    ai_action: "Формулировать как 'может рассматриваться'"

  C — Низкий:
    sources: "Серии случаев, case-control исследования"
    certainty: "Ограниченная уверенность"
    ai_action: "Формулировать как 'в некоторых источниках упоминается'"

  D — Очень низкий:
    sources: "Единичные наблюдения, case reports"
    certainty: "Очень низкая уверенность"
    ai_action: "Обязательно предупредить о низкой доказательности"

  E — Экспертное мнение:
    sources: "Консенсус экспертов без исследований"
    certainty: "Основано на опыте, а не исследованиях"
    ai_action: "Указать 'экспертное мнение, не подтверждено исследованиями'"
```

## Интерпретация лабораторных анализов

### Принципы

```yaml
Правила:
  - Сравнивать с референсными значениями (возраст, пол)
  - Учитывать динамику (если есть предыдущие результаты)
  - Выявлять паттерны отклонений (группы показателей)
  - НЕ ставить диагноз по одному показателю
  - Всегда рекомендовать интерпретацию врачом

Критические значения (немедленная эскалация):
  - Глюкоза: <2.8 или >30 ммоль/л
  - Калий: <2.5 или >6.5 ммоль/л
  - Натрий: <120 или >160 ммоль/л
  - Гемоглобин: <60 или >200 г/л
  - Тромбоциты: <20 или >1000 ×10⁹/л
  - Креатинин: >800 мкмоль/л
  - Тропонин: значительное повышение
```

### Паттерны отклонений

```yaml
Воспалительный паттерн:
  indicators: [лейкоциты↑, СОЭ↑, СРБ↑]
  suggests: "Воспалительный процесс"
  differential: [инфекция, аутоиммунное, онкология]

Анемический паттерн:
  indicators: [гемоглобин↓, эритроциты↓, гематокрит↓]
  subtypes:
    железодефицитная: [ферритин↓, железо↓, ОЖСС↑, MCV↓]
    В12-дефицитная: [MCV↑, В12↓]
    хроническая: [ферритин↑, железо↓, ОЖСС↓]

Печёночный паттерн:
  indicators: [АЛТ↑, АСТ↑, билирубин↑]
  subtypes:
    цитолиз: [АЛТ>>АСТ]
    холестаз: [ЩФ↑, ГГТ↑, прямой билирубин↑]
    гепатоцеллюлярная_недостаточность: [альбумин↓, ПТИ↓]

Почечный паттерн:
  indicators: [креатинин↑, мочевина↑, СКФ↓]
  staging: "По CKD-EPI формуле (см. skills/medical-api-integration.md)"

Тиреоидный паттерн:
  гипотиреоз: [ТТГ↑, Т4↓]
  гипертиреоз: [ТТГ↓, Т4↑]
  субклинический: [ТТГ отклонён, Т4 в норме]
```

## Проверка лекарственных взаимодействий

```yaml
Workflow:
  1. Получить список текущих лекарств пациента
  2. Для каждой пары проверить через RxNorm API
  3. Классифицировать interaction severity:
     - Major (серьёзное) → предупредить, рекомендовать консультацию врача
     - Moderate (умеренное) → информировать
     - Minor (незначительное) → упомянуть при необходимости
  4. Проверить противопоказания при текущих диагнозах
  5. Логировать: drug_interaction_detected

Источники:
  - DrugBank
  - RxNorm (NLM)
  - OpenFDA adverse events
  - Клинические рекомендации Минздрав
```

## Безопасность рассуждений

### Запрещённые паттерны

```yaml
НИКОГДА не генерировать:
  - Утвердительный диагноз: "У вас [диагноз]"
  - Назначение лечения: "Принимайте [лекарство]"
  - Исключение серьёзного: "Это точно не [опасное заболевание]"
  - Ложное успокоение: "Ничего страшного, не беспокойтесь"
  - Прогноз: "Вы выздоровеете через [срок]"

ВСЕГДА включать:
  - Дифференциальный диагноз (минимум 2-3 варианта)
  - Уровень уверенности для каждой гипотезы
  - Рекомендацию обратиться к врачу
  - Соответствующий дисклеймер
  - Red flag screening на каждом шаге
```

### Fallback стратегия

```yaml
При неопределённости (все confidence < 30%):
  1. Честно сообщить о невозможности определить причину
  2. Рекомендовать обратиться к терапевту/врачу общей практики
  3. Предложить базовые обследования (ОАК, ОАМ, биохимия)
  4. Поставить дисклеймер типа 'general'

При противоречивых симптомах:
  1. Перечислить возможные объяснения
  2. Запросить уточняющую информацию
  3. Рекомендовать очную консультацию врача
```

## Качество клинического рассуждения

### Метрики

```yaml
Accuracy metrics:
  - Top-1 accuracy: % случаев, когда правильный диагноз на первом месте
  - Top-3 accuracy: % случаев, когда правильный диагноз в top-3
  - Red flag sensitivity: >95% (false negative < 5%)
  - Red flag specificity: >70%
  - Inappropriate confidence: < 5% (high confidence при неправильном диагнозе)

Safety metrics:
  - Missed red flags: 0 (целевой показатель)
  - Forbidden formulations: 0
  - Missing disclaimers: 0
  - Escalation response time: < 1 секунда
```
