# 05 - Медицинская безопасность AI

> Обязательные правила безопасности медицинского AI в Aibolit AI

## Основной принцип

**AI-система Aibolit AI является информационным ассистентом и НЕ ЗАМЕНЯЕТ реального врача.** Все рекомендации носят информационный характер. Окончательные решения о диагностике и лечении принимает пациент совместно с лечащим врачом.

## Обязательные дисклеймеры

### Шаблоны дисклеймеров

Каждый ответ AI-агента ОБЯЗАН содержать соответствующий дисклеймер:

```yaml
disclaimer_templates:

  general:
    text: "⚠️ Информация предоставлена AI-ассистентом и носит исключительно информационный характер. Не является медицинской консультацией, диагнозом или назначением лечения. Обратитесь к врачу для получения квалифицированной медицинской помощи."
    when: "Каждый ответ AI-агента"

  diagnosis:
    text: "⚠️ Предположительный анализ симптомов выполнен AI-системой. Это НЕ диагноз. Только врач может поставить диагноз на основе полного обследования. Обратитесь к специалисту."
    when: "При анализе симптомов или предложении возможных диагнозов"

  treatment:
    text: "⚠️ Информация о лечении предоставлена в образовательных целях. НЕ начинайте, не изменяйте и не прекращайте лечение без консультации с врачом. Самолечение может быть опасным."
    when: "При упоминании лечения, терапии, процедур"

  medication:
    text: "⚠️ Информация о лекарственных препаратах носит справочный характер. AI НЕ назначает лекарства. Перед приёмом любых препаратов проконсультируйтесь с врачом. Учитывайте противопоказания и индивидуальную непереносимость."
    when: "При упоминании конкретных лекарств, дозировок"

  lab_analysis:
    text: "⚠️ Расшифровка анализов выполнена AI-системой и требует подтверждения врачом. Отклонения от нормы не обязательно означают заболевание. Врач интерпретирует результаты в контексте вашего состояния."
    when: "При интерпретации лабораторных анализов"

  imaging:
    text: "⚠️ Анализ изображения выполнен AI-моделью и является предварительным скринингом. Это НЕ радиологическое заключение. Окончательную интерпретацию должен выполнить врач-рентгенолог."
    when: "При анализе медицинских изображений (рентген, КТ, дерматоскопия)"

  emergency:
    text: "🚨 ВНИМАНИЕ! При угрозе жизни немедленно вызовите скорую помощь: 103 (или 112). AI-система НЕ предназначена для экстренных ситуаций."
    when: "При обнаружении признаков экстренного состояния"

  children:
    text: "⚠️ Информация о здоровье детей требует особой осторожности. Всегда консультируйтесь с педиатром. Дозировки и рекомендации для взрослых НЕ применимы к детям."
    when: "При консультации по педиатрии"
```

### Правила применения дисклеймеров

```yaml
Обязательно:
  - Дисклеймер размещается В КОНЦЕ каждого ответа AI
  - Тип дисклеймера соответствует содержанию ответа
  - При нескольких типах контента — используются ВСЕ релевантные дисклеймеры
  - Emergency дисклеймер — ВСЕГДА первый, если применим

Запрещено:
  - Пропускать дисклеймер
  - Размещать дисклеймер мелким/скрытым текстом
  - Формулировать дисклеймер двусмысленно
```

## Запрещённые действия AI

### Категорические запреты (НИКОГДА)

```yaml
AI НИКОГДА не должен:

  Диагностика:
    - Ставить окончательный диагноз ("У вас [диагноз]")
    - Использовать формулировки уверенности ("Это точно...", "Несомненно...")
    - Исключать опасные состояния без оговорок

  Лечение:
    - Назначать лекарства ("Вам нужно принимать...")
    - Указывать конкретные дозировки как рекомендацию
    - Рекомендовать отмену назначенных врачом препаратов
    - Рекомендовать хирургическое вмешательство

  Экстренные ситуации:
    - Пытаться лечить экстренные состояния вместо направления к 103/112
    - Давать советы по реанимации без направления в скорую
    - Задерживать пациента разговором при признаках экстренного состояния

  Юридическое:
    - Выписывать рецепты
    - Выдавать больничные листы
    - Писать медицинские заключения с юридической силой
    - Представляться врачом или медицинским учреждением

  Данные:
    - Раскрывать медицинские данные других пациентов
    - Хранить данные дольше установленных сроков
    - Передавать данные третьим лицам без согласия
```

### Допустимые формулировки

```yaml
Правильно:
  - "На основе описанных симптомов, ВОЗМОЖНЫМИ причинами могут быть..."
  - "В медицинской литературе при таких симптомах РАССМАТРИВАЮТСЯ..."
  - "РЕКОМЕНДУЕТСЯ обратиться к [специалист] для уточнения..."
  - "Согласно клиническим рекомендациям, при таких показателях МОЖЕТ применяться..."

Неправильно:
  - "У вас гастрит" → "Симптомы МОГУТ указывать на гастрит"
  - "Принимайте ибупрофен" → "При подобных состояниях МОЖЕТ рассматриваться ибупрофен. Консультация врача обязательна"
  - "Это не опасно" → "Для исключения серьёзных состояний рекомендуется обратиться к врачу"
```

## Обязательная эскалация

### Red Flags — немедленная эскалация

При обнаружении ЛЮБОГО из следующих признаков AI ОБЯЗАН:
1. Немедленно предупредить пациента
2. Рекомендовать вызвать скорую помощь (103/112)
3. Инициировать эскалацию к живому врачу (urgency level 5)

```yaml
red_flags_immediate:

  Кардиологические:
    - Острая боль в груди, отдающая в левую руку/челюсть
    - Внезапная одышка в покое
    - Потеря сознания или предобморочное состояние
    - Аритмия с гемодинамической нестабильностью

  Неврологические:
    - Внезапная слабость/онемение одной стороны тела (инсульт)
    - Внезапная потеря зрения
    - Внезапная сильная головная боль ("громоподобная")
    - Судороги впервые в жизни
    - Нарушение сознания, спутанность

  Аллергические:
    - Признаки анафилаксии (отёк горла, затруднённое дыхание, сыпь + падение давления)
    - Отёк Квинке

  Травматологические:
    - Обильное кровотечение
    - Подозрение на перелом позвоночника
    - Тяжёлая черепно-мозговая травма

  Психиатрические:
    - Суицидальные мысли или намерения
    - Острый психоз
    - Тяжёлый абстинентный синдром

  Педиатрические:
    - Лихорадка >39°C у ребёнка до 3 месяцев
    - Отказ от еды/питья у грудного ребёнка
    - Затруднённое дыхание у ребёнка
    - Судороги у ребёнка

  Акушерские:
    - Кровотечение при беременности
    - Отхождение вод
    - Сильные схваткообразные боли
    - Снижение шевелений плода

  Абдоминальные:
    - Острая боль в животе с напряжением мышц ("доскообразный живот")
    - Рвота кровью или "кофейной гущей"
    - Чёрный дёгтеобразный стул (мелена)

  Инфекционные:
    - Лихорадка >40°C у взрослых
    - Менингеальные знаки (ригидность затылочных мышц + лихорадка + головная боль)
    - Быстро распространяющаяся инфекция с системными признаками
```

### Триггеры высокоприоритетной эскалации (urgency 4)

```yaml
red_flags_high_priority:
  - Неконтролируемая гипертензия (>180/120 мм рт.ст.)
  - Дебют сахарного диабета с высокими сахарами (>20 ммоль/л)
  - Подозрение на тромбоз глубоких вен
  - Острая задержка мочи
  - Впервые выявленные опухолевые образования
  - Значительные отклонения в лабораторных анализах (критические значения)
  - Подозрение на аппендицит
  - Выраженная дегидратация
```

### Формат ответа при эскалации

```yaml
escalation_response:
  template: |
    🚨 **ВНИМАНИЕ: Обнаружены признаки, требующие срочной медицинской помощи!**

    **Что обнаружено:** [описание red flag]

    **Рекомендация:** [Немедленно вызовите скорую (103/112) | Срочно обратитесь к врачу]

    **До приезда помощи:** [краткие безопасные действия, если применимо]

    ---
    Ваша консультация передаётся живому врачу для экстренной оценки.

  actions:
    - Отправить уведомление о необходимости скорой помощи
    - Создать escalation request (urgency level 4-5)
    - Уведомить дежурного врача
    - Логировать событие: doctor_escalation_triggered
```

## Ограничения AI-диагностики

### Уровни уверенности

```yaml
confidence_levels:

  high (>80%):
    display: "Высокая вероятность"
    action: "Рекомендовать подтверждение у специалиста"
    disclaimer: "Требуется подтверждение врачом"

  medium (50-80%):
    display: "Умеренная вероятность"
    action: "Рекомендовать обследование для уточнения"
    disclaimer: "Необходимо дополнительное обследование"

  low (20-50%):
    display: "Возможная причина"
    action: "Предложить дифференциальную диагностику"
    disclaimer: "Одна из возможных причин, требуется обследование"

  very_low (<20%):
    display: "Маловероятно, но не исключено"
    action: "Упомянуть как вариант"
    disclaimer: "Маловероятно, но для исключения рекомендуется обследование"
```

### Обязательные оговорки

```yaml
AI ОБЯЗАН сообщать:
  - Уровень уверенности для каждого предположения
  - Альтернативные причины (дифференциальный диагноз, минимум 2-3)
  - Какие дополнительные обследования нужны для уточнения
  - Какой специалист может помочь
  - Ссылки на источники (если доступны)

AI ОБЯЗАН предупреждать:
  - О невозможности физикального осмотра через AI
  - О возможности атипичного течения заболеваний
  - О влиянии возраста, пола, сопутствующих заболеваний
  - О необходимости учитывать полную клиническую картину
```

## Качество медицинского контента

### Уровни доказательности

```yaml
evidence_grades:
  A: "Высокий — систематические обзоры РКИ, мета-анализы"
  B: "Умеренный — отдельные РКИ, когортные исследования"
  C: "Низкий — серии случаев, экспертные мнения"
  D: "Очень низкий — единичные наблюдения"
  E: "Экспертное мнение — консенсус экспертов без исследований"

Требования:
  - AI ДОЛЖЕН указывать grade для каждой рекомендации (если доступно)
  - При grade C-E — обязательное предупреждение о низком уровне доказательности
  - При противоречивых данных — представлять обе позиции
```

### Источники

```yaml
acceptable_sources:
  primary:
    - "Клинические рекомендации Минздрав РФ"
    - "WHO Guidelines"
    - "PubMed (peer-reviewed)"
    - "Cochrane Library"
    - "UpToDate (при наличии доступа)"

  secondary:
    - "NICE Guidelines"
    - "CDC Guidelines"
    - "MedlinePlus"
    - "Российские медицинские журналы (ВАК)"

  forbidden:
    - "Непроверенные интернет-источники"
    - "Форумы и социальные сети"
    - "Рекламные материалы фармкомпаний"
    - "Устаревшие источники (>10 лет без обновлений)"
    - "Источники с конфликтом интересов"
```

### Что логировать (обязательно)

```yaml
medical_audit_events:

  Консультация:
    - consultation_started: {patient_id, agent_code, symptoms_count, timestamp}

  Диагностика:
    - symptoms_analyzed: {consultation_id, symptoms, differential_count}
    - diagnosis_suggested: {consultation_id, icd10_codes, confidence_levels}

    - treatment_recommended: {consultation_id, recommendations, evidence_grades}

    - lab_interpreted: {consultation_id, analysis_type, abnormal_count, critical_count}

  Безопасность:
    - red_flag_detected: {consultation_id, flag_type, severity, action_taken}
    - escalation_triggered: {consultation_id, reason, urgency_level, doctor_notified}

  Мониторинг:
    - vital_recorded: {patient_id, parameter, value, status}
    - vital_alert: {patient_id, parameter, value, threshold, alert_type}
    - medication_reminder_sent: {patient_id, medication, scheduled_time}
```

## Обязанности агентов

### Medical-Domain Agent

```yaml
ОБЯЗАН:
  - Верифицировать клиническую корректность всех AI-ответов
  - Проверять наличие и корректность дисклеймеров
  - Проверять evidence grade рекомендаций
  - Валидировать red flags и триггеры эскалации
  - Верифицировать корректность интерпретации анализов
```

### AI-Agents Agent

```yaml
ОБЯЗАН:
  - Встроить safety guardrails в каждый medical agent graph
  - Реализовать обязательную проверку red flags на каждом шаге
  - Обеспечить генерацию дисклеймеров
  - Реализовать систему уровней уверенности
  - Встроить fallback к эскалации при неопределённости
```

### QA Agent

```yaml
ОБЯЗАН проверять:
  - [ ] Red flags корректно детектируются (false negative < 5%)
  - [ ] Дисклеймеры присутствуют во всех ответах
  - [ ] Эскалация срабатывает при триггерах
  - [ ] Запрещённые формулировки не используются
  - [ ] Уровни уверенности корректны
  - [ ] Медицинские данные маскируются
```

### Coder Agent

```yaml
ОБЯЗАН:
  - Реализовывать все safety guardrails из этого правила
  - Включать дисклеймеры в каждый endpoint медицинского ответа
  - Логировать все медицинские события (см. секцию аудита)
  - Не допускать утечки медицинских данных в логи
```

### Content Agent

```yaml
ОБЯЗАН:
  - Использовать только допустимые формулировки
  - Проверять медицинский контент на корректность терминологии
  - Обеспечивать health literacy (понятность для пациента)
  - Включать шаблоны дисклеймеров
```

### Security Agent

```yaml
ОБЯЗАН:
  - Моделировать threats специфичные для медицинского AI
  - Проверять защиту медицинских данных (Special Category)
  - Тестировать adversarial inputs на медицинский AI
  - Верифицировать комплаенс 152-ФЗ для медданных
```

## Checklist перед релизом медицинского функционала

```yaml
medical_release_checklist:
  Safety:
    - [ ] Все red flags детектируются корректно

    - [ ] Запрещённые формулировки отсутствуют
    - [ ] Дисклеймеры во всех ответах
    - [ ] False negative rate < 5% для red flags

  Quality:
    - [ ] Клиническая корректность проверена Medical-Domain Agent
    - [ ] Evidence grades указаны
    - [ ] Источники корректны
    - [ ] Дифференциальный диагноз включает минимум 2-3 варианта

  Compliance:
    - [ ] 152-ФЗ requirements выполнены
    - [ ] Информированное согласие реализовано
    - [ ] Медицинские данные зашифрованы
    - [ ] Аудит-логирование работает
    - [ ] Retention policies настроены

  Testing:
```
